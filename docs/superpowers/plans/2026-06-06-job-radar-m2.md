# Job Radar M2 Implementation Plan (coverage + scale)

**Goal:** Turn M1's working pipeline into real coverage. Add Workday + SmartRecruiters adapters (5 ATSs total), a bulk seed importer to scale the watch-list to thousands, retry/backoff for robustness, and concurrent LLM scoring for speed.

**Builds on:** `2026-06-01-job-radar-m1.md` (shipped, 34 tests green, verified live).

**Conventions:** same as M1. TDD, exact paths, tests stay green, commit per logical unit, Co-Authored-By trailer.

---

## Task list

### Task A: Retry/backoff in the HTTP layer
- Modify `src/job_radar/adapters/base.py`: `get_json` retries on 429/5xx with capped exponential backoff, honoring `Retry-After`.
- New `tests/test_base.py`: covers `to_dt`/`from_ms`/`strip_html` (previously untested) + a retry test (transport fails 503 once, then 200; `asyncio.sleep` monkeypatched to no-op).

### Task B: Workday adapter
- Create `src/job_radar/adapters/workday.py`: per-tenant CXS POST API, paginated. Uses `company.wd_host` + `company.wd_site`. Relative `postedOn` parsed best-effort. Description left empty (Workday needs a 2nd call; scores on title+location for M2).
- `tests/fixtures/workday_jobs.json` + `tests/test_adapters_workday.py` (parse + relative-date + fetch via MockTransport).

### Task C: SmartRecruiters adapter
- Create `src/job_radar/adapters/smartrecruiters.py`: public `api.smartrecruiters.com/v1/companies/{slug}/postings`, paginated. Structured location.
- `tests/fixtures/smartrecruiters_jobs.json` + `tests/test_adapters_smartrecruiters.py`.

### Task D: Wire new adapters in
- Modify `src/job_radar/sources.py`: add `workday` + `smartrecruiters` to `ADAPTERS`.

### Task E: Bulk seed importer
- Create `src/job_radar/seed.py`: `parse_line` + `merge` + `main`. Run via `python -m job_radar.seed <list.csv> [config/companies.yaml]`. Format per line: `slug,ats[,tier]`; `#` comments; dedups against existing.
- `tests/test_seed.py`: parse_line (valid/invalid/comment), merge (dedup + add).

### Task F: Expanded starter watch-list
- Modify `config/companies.yaml`: curated multi-ATS set (best-effort slugs; auto-skip on 404). Workday entries carry `wd_host` + `wd_site`.

### Task G: Concurrent LLM scoring
- Modify `src/job_radar/pipeline.py`: score survivors concurrently via a bounded semaphore (`_score_all`), preserving order, then route. Cuts wall-clock when many survivors.

### Task H: Docs
- Update `README.md` (5 ATSs, seed importer usage). This plan committed alongside.

## Notes / decisions
- **Workday tenants are data, not code.** Exact `tenant`/`wd_host`/`wd_site` strings are error-prone; the adapter is the deliverable. `companies.yaml` documents how to find them (open careers page, watch the `/wday/cxs/.../jobs` request).
- **Empty descriptions** for Workday + SmartRecruiters (M2): the LLM scores on title + company + location. Detail-fetch is M3.
- **Durable store (Supabase)** stays M3; JSON-in-Actions-cache is fine until the watch-list is large.
